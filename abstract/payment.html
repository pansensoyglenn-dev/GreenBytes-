The Abstract Payment Gateway

Decoupling your application from PayPal, GCash, Maya, Stripe, and more.

üìò Project Overview: Building Change-Resilient GreenBytes Systems

Payment providers come and go, change fees, update APIs, or get replaced by regional favorites. Hard-coding your checkout logic to one specific gateway (e.g. only PayPal) creates tight coupling ‚Äî every new integration or provider change requires painful refactoring across your entire codebase.

Using Python's abc module, we define a clean abstraction layer ‚Äî a strict contract that every payment gateway must follow. This lets your business logic (cart ‚Üí checkout ‚Üí payment) remain completely agnostic to which gateway is actually processing the transaction.

Uniform API

One consistent .authorize() + .transfer() interface ‚Äî no matter the provider.

Encapsulated Complexity

API keys, OAuth flows, OTP logic, error handling, currency conversion ‚Äî all hidden inside concrete classes.

Easy to Extend

New gateway? Just create one new subclass. No changes to checkout or business logic.

Why Choose Abstraction for Financial Tech?

In modern web development, scalability is everything. If your e-commerce platform relies on Stripe today, but you want to expand to the Philippine market tomorrow, you will need to integrate GCash or Maya. Without Abstraction, you would have if/else statements scattered through your code.

By using the Abstract Base Class (ABC) pattern shown below, your system becomes "Closed for modification but Open for extension." You add new features by adding new files, not by breaking existing ones.

Deep Dive: 6 Key Mechanics of Abstraction

1. Formalizing the "Contract"

The PaymentGateway(ABC) acts as a legal document for your code. It ensures that any team member creating a new payment method follows the same method names and arguments, preventing integration bugs before they happen.

2. Prevention of Instantiation

By inheriting from ABC, Python prevents you from creating a generic "PaymentGateway" object. This is a safety feature: a payment gateway is a concept, not a real tool. Only specific ones like PayPal should exist as objects.

3. Mandatory Implementation

The @abstractmethod decorator acts as a compiler-level check. If a child class (like Maya) forgets to include the transfer() method, Python will refuse to run, forcing the developer to complete the required logic immediately.

4. Decoupled Business Logic

Your process_checkout function only cares about the concept of a gateway. This means you can swap the entire financial backend of your website without changing a single line of your checkout UI or database logic.

5. Encapsulated Vendor Secrets

Each class (PayPal, GCash) hides its own API keys and specific connection protocols. The rest of the app doesn't need to know how PayPal handles OAuth or how GCash handles OTPs; it just calls .authorize().

6. Polymorphic Flexibility

Abstraction enables polymorphism, allowing a single list of transactions to process multiple different types of payment objects in a single loop, drastically reducing the amount of redundant if/else code in your system.

1. Core Abstraction Pattern

The central idea: your application never talks directly to PayPal, GCash, Stripe, etc. It only talks to the PaymentGateway interface.

from abc import ABC, abstractmethod

class PaymentGateway(ABC):
    """Abstract base class defining the payment gateway contract."""
    
    @abstractmethod
    def authorize(self, amount: float) -> None:
        """Authorize / reserve the amount (e.g. 3D Secure, OTP, card check)."""
        pass

    @abstractmethod
    def transfer(self) -> None:
        """Finalize / capture the payment."""
        pass


class PayPal(PaymentGateway):
    def authorize(self, amount: float) -> None:
        print(f"--- [PayPal] Authorizing ${amount:.2f} USD (OAuth flow) ---")
    
    def transfer(self) -> None:
        print("--- [PayPal] Funds captured successfully ---")


class GCash(PaymentGateway):
    def authorize(self, amount: float) -> None:
        print(f"--- [GCash] Sending OTP for ‚Ç±{amount:.2f} ---")
    
    def transfer(self) -> None:
        print("--- [GCash] GCash wallet debited ---")


class BankTransfer(PaymentGateway):
    def authorize(self, amount: float) -> None:
        print(f"--- [BankTransfer] Verifying account & initiating hold for ${amount:.2f} ---")
    
    def transfer(self) -> None:
        print("--- [BankTransfer] Wire transfer scheduled ---")


def process_checkout(gateway: PaymentGateway, amount: float) -> None:
    """Core checkout logic ‚Äî completely gateway-agnostic."""
    print(f"[Checkout] Processing payment of {amount:.2f}...")
    gateway.authorize(amount)
    gateway.transfer()
    print("[Checkout] Payment completed successfully.\n")


# Demo
print("=== SYSTEM VERIFICATION ===\n")
process_checkout(PayPal(), 150.00)
process_checkout(GCash(), 5000.00)
process_checkout(BankTransfer(), 250.00)
print("=== VERIFICATION COMPLETE ===")
