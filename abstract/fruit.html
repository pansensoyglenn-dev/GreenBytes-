The Preface

The Paradigm of Abstraction in Modern Software

In the evolving landscape of computational logic, the ability to decouple high-level business requirements from low-level implementation details is the hallmark of professional software engineering. This tutorial explores The Fruit Hub Framework, a sophisticated retail management model built upon the pillars of Object-Oriented Programming (OOP). By utilizing Python’s Abstract Base Classes (ABC), we demonstrate how to create a scalable, modular environment that mirrors real-world retail operations.

from abc import ABC, abstractmethod
from datetime import datetime
from typing import List
from dataclasses import dataclass

class FruitItem(ABC):
    """Abstract interface for all sellable fruit items."""
    @abstractmethod
    def get_name(self) -> str: pass
    @abstractmethod
    def get_price_per_kg(self) -> float: pass
    @abstractmethod
    def get_unit(self) -> str: pass
    def calculate_subtotal(self, quantity: float) -> float:
        if quantity <= 0:
            raise ValueError("Quantity must be positive")
        return round(self.get_price_per_kg() * quantity, 2)
    def __str__(self) -> str:
        return f"{self.get_name()} (₱{self.get_price_per_kg():.2f}/{self.get_unit()})"

class ConcreteFruit(FruitItem):
    """Concrete implementation using composition over one-class-per-fruit."""
    def __init__(self, name: str, price_per_kg: float, unit: str = "kg"):
        self._name = name
        self._price = price_per_kg
        self._unit = unit
    def get_name(self) -> str: return self._name
    def get_price_per_kg(self) -> float: return self._price
    def get_unit(self) -> str: return self._unit

class ShoppingCart:
    """Manages cart items with type-safe composition."""
    def __init__(self): self.items: List[CartLine] = []
    def add_item(self, fruit: FruitItem, kilograms: float):
        subtotal = fruit.calculate_subtotal(kilograms)
        self.items.append(CartLine(fruit=fruit, kg=kilograms, subtotal=subtotal))
        print(f"→ Added {kilograms:.2f} kg {fruit.get_name()} → ₱{subtotal:,.2f}")
    def get_total(self) -> float:
        return round(sum(item.subtotal for item in self.items), 2)
    def print_receipt(self):
        print("\n" + "═" * 60)
        print(f"      FRUIT STAND RECEIPT – {datetime.now().strftime('%Y-%m-%d %I:%M %p')}")
        print("═" * 60)
        for item in self.items:
            f = item.fruit
            print(f"{f.get_name():<15} {item.kg:>6.2f} kg   ₱{item.subtotal:>10,.2f}")
        print("─" * 60)
        print(f"TOTAL TO PAY              ₱{self.get_total():>10,.2f}")
        print("═" * 60)

@dataclass
class CartLine:
    fruit: FruitItem
    kg: float
    subtotal: float

def main():
    catalog = {
        "1": ConcreteFruit("Durian", 120.0),
        "2": ConcreteFruit("Orange", 80.0),
        "3": ConcreteFruit("Apple", 80.0),
        "4": ConcreteFruit("Lanzones", 70.0),
        "5": ConcreteFruit("Mangosteen", 75.0),
        "6": ConcreteFruit("Guava", 80.0),
        "7": ConcreteFruit("Guyabano", 80.0),
        "8": ConcreteFruit("Banana", 60.0),
        "9": ConcreteFruit("Grapes", 120.0),
    }
    cart = ShoppingCart()
    
    print("Welcome to Polomolok Fresh Fruits!")
    print("\n" + "─" * 30)
    print(f"{'ID':<4} {'FRUIT':<15} {'PRICE/kg':<10}")
    print("─" * 30)
    for key, fruit in catalog.items():
        print(f"{key:<4} {fruit.get_name():<15} ₱{fruit.get_price_per_kg():>6.2f}")
    print("─" * 30)

    while True:
        choice = input("\nEnter fruit ID (0 to finish): ").strip()
        if choice == "0": break
        if choice not in catalog:
            print("→ Item not found. Refer to ID list.")
            continue
        try:
            kg = float(input(f"Weight for {catalog[choice].get_name()} (kg): "))
            if kg > 0: cart.add_item(catalog[choice], kg)
            else: print("→ Quantity must be positive.")
        except ValueError: print("→ Invalid input. Enter numerical weight.")
            
    if cart.items: cart.print_receipt()

if __name__ == "__main__":
    main()

02. Theoretical Depth

Why Major Elements Were Used

Import Statements: These provide essential tools for abstraction and type safety. ABC and abstractmethod enable formal abstraction. datetime is for timestamping receipts, making the program realistic for business use. List and Dict add type hints, improving code readability and IDE support.

Abstract Base Class: FruitItem(ABC)

Why used: This is the core of abstraction. It defines an interface (contract) that all fruits must follow, hiding implementation details while ensuring consistency.

Structural Logic Mapping

It promotes polymorphism and makes the program extensible—new fruits can be added without changing the cart or main logic. By enforcing method overrides in subclasses, it ensures that every fruit has a name, price, and unit before the code can even run.

03. Operational Pillars

12 Key Pillars of Operation

01
Import Statements Operation
When the script runs, Python loads the abc module for abstraction and datetime for current time access. These foundational imports establish the infrastructure for formal interface contracts (via ABC) and realistic business timestamping on receipts.
02
Abstract Base Class: FruitItem
Defines the blueprint. You can't instantiate it directly. It ensures subclasses follow the mandatory method structure and enforces a strict contract across all fruit types.
03
Abstract Method Placeholder
The pass keyword acts as a syntactic requirement, forcing child classes to implement logic. This placeholder triggers Python's ABC enforcement mechanism.
04
Concrete Method: calculate_subtotal
Multiplies quantity by price dynamically fetched via polymorphism. Located in the base class, this method provides reusable default behavior that every fruit inherits automatically.
05
The str Method Logic
Automatically formats the fruit object into a human-readable menu string. Because __str__ relies on the abstract interface, it remains polymorphic for any FruitItem subclass.
06
Concrete Fruit Classes
Specific implementations (Durian, Orange, etc.) that fill in the "price" and "name" details of the blueprint. These classes are deliberately minimal to keep them focused.
07
ShoppingCart Composition
Maintains a stateful list of transaction records. By composing FruitItem instances, the cart stays decoupled from concrete fruit details.
08
Catalog Dictionary Lookup
Provides high-efficiency mapping between numeric user choices and fruit object instances, enabling fast O(1) lookups.
09
Interactive Loop Workflow
Manages the UI experience, taking orders until the exit signal (0) is triggered. This creates a simple flow that matches a real fruit sale.
10
Exception Handling Mechanism
Uses try/except to prevent user input errors from crashing the system, ensuring the program stays alive during real use.
11
Receipt and Change Logic
Finalizes the session by summing subtotals and printing a formatted, timestamped transaction record using Unicode characters.
12
The main Guard
Ensures script integrity by preventing the interactive loop from running if the file is imported elsewhere, a fundamental Python best practice.
