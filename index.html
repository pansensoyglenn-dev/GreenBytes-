The Logic of Encapsulation

Encapsulation in Python involves bundling data (attributes) and methods that operate on the data within a class, while restricting direct access to some components (e.g., using private variables with underscores). This promotes data hiding and controlled access via getter and setter methods. Below are 10 sample programs, each with an algorithmic narrative and step-by-step explanation.

Algorithmic Narrative: Create a class to manage a bank account where the balance is private. Initialize with a balance, provide methods to deposit (add amount if positive), withdraw (subtract if sufficient funds), and get balance. Ensure direct access to balance is restricted.

class BankAccount:
    def __init__(self, initial_balance=0):
        self.__balance = initial_balance  # Private attribute
    def deposit(self, amount):
        if amount > 0: self.__balance += amount
    def withdraw(self, amount):
        if 0 < amount <= self.__balance: self.__balance -= amount
    def get_balance(self): return self.__balance

account = BankAccount(100)
account.deposit(50)
account.withdraw(30)
print(account.get_balance()) # Output: 120


Step-by-Step: 1. Define class. 2. Use __ for private balance. 3. Validate amounts in methods. 4. Use getter for access.

Algorithmic Narrative: Design a class for employee details with private salary. Initialize with name and salary, provide a method to give raise (increase by percentage), and a getter for salary. Prevent negative raises.

class Employee:
    def __init__(self, name, salary):
        self.name = name
        self.__salary = salary
    def give_raise(self, percentage):
        if percentage > 0: self.__salary += self.__salary * (percentage / 100)
    def get_salary(self): return self.__salary

emp = Employee("Alice", 50000)
emp.give_raise(10)
print(emp.get_salary()) # Output: 55000.0

Algorithmic Narrative: Model a car with private fuel level. Initialize with fuel, methods to drive (reduce fuel based on distance), refuel (add fuel), and get fuel level. Ensure fuel doesn't go negative.

class Car:
    def __init__(self, fuel=50):
        self.__fuel = fuel
    def drive(self, distance):
        fuel_needed = distance / 10
        if fuel_needed <= self.__fuel: self.__fuel -= fuel_needed
    def refuel(self, amount):
        if amount > 0: self.__fuel += amount
    def get_fuel(self): return self.__fuel

my_car = Car(); my_car.drive(200); my_car.refuel(30)
print(my_car.get_fuel()) # Output: 60.0

The Logic of Abstraction

Abstraction in Python hides complex implementation details and exposes only essential features, often using abstract base classes (ABCs). Below are 10 samples with step-by-step explanations.

from abc import ABC, abstractmethod
class Shape(ABC):
    @abstractmethod
    def area(self): pass
class Circle(Shape):
    def __init__(self, r): self.r = r
    def area(self): return 3.14 * self.r ** 2

class Payment(ABC):
    @abstractmethod
    def process(self, amount): pass
class CreditCard(Payment):
    def process(self, amount): return f"CC Processed {amount}"

class Animal(ABC):
    @abstractmethod
    def make_sound(self): pass
class Dog(Animal):
    def make_sound(self): return "Woof"

class Vehicle(ABC):
    @abstractmethod
    def start(self): pass
class Bike(Vehicle):
    def start(self): return "Bike kickstart"

class Database(ABC):
    @abstractmethod
    def connect(self): pass
class MySQL(Database):
    def connect(self): return "MySQL Connected"

class FileHandler(ABC):
    @abstractmethod
    def read(self): pass
class TextFile(FileHandler):
    def read(self): return "Reading Text..."

class Notification(ABC):
    @abstractmethod
    def send(self, msg): pass
class Email(Notification):
    def send(self, msg): return f"Email: {msg}"

class Logger(ABC):
    @abstractmethod
    def log(self, msg): pass
class ConsoleLogger(Logger):
    def log(self, msg): print(f"Log: {msg}")

class Encryptor(ABC):
    @abstractmethod
    def encrypt(self, data): pass
class AES(Encryptor):
    def encrypt(self, data): return f"AES Encrypted {data}"

class Media(ABC):
    @abstractmethod
    def play(self): pass
class Video(Media):
    def play(self): return "Video Playing"

The Logic of Inheritance

Inheritance allows a subclass to inherit attributes and methods from a base class, promoting code reuse. Below are 10 samples with step-by-step narratives.

class Animal:
    def eat(self): return "Eating"
class Dog(Animal):
    def bark(self): return "Woof"

class Shape:
    def __init__(self, color): self.color = color
class Rectangle(Shape):
    def __init__(self, color, l, w):
        super().__init__(color)
        self.l, self.w = l, w
    def area(self): return self.l * self.w

class Person:
    def __init__(self, name): self.name = name
class Student(Person):
    def __init__(self, name, grade):
        super().__init__(name); self.grade = grade
    def study(self): return f"{self.name} is studying"

class Vehicle:
    def __init__(self, speed): self.speed = speed
    def accelerate(self): self.speed += 10
class Car(Vehicle):
    def accelerate(self): self.speed += 20

class Employee:
    def __init__(self, salary): self.salary = salary
class Manager(Employee):
    def __init__(self, salary, team):
        super().__init__(salary); self.team = team
    def bonus(self): return self.salary * 0.1 * self.team

class Bird:
    def fly(self): return "Flying"
class Eagle(Bird):
    def hunt(self): return "Hunting"

class Account:
    def __init__(self, balance): self.balance = balance
class SavingsAccount(Account):
    def add_interest(self, rate): self.balance += self.balance * (rate / 100)

class Device:
    def power_on(self): return "Power On"
class Smartphone(Device):
    def call(self): return "Calling..."

class Fruit:
    def __init__(self, color): self.color = color
class Apple(Fruit):
    def __init__(self, color, variety):
        super().__init__(color); self.variety = variety

class Tool:
    def use(self): return "Using tool"
class Hammer(Tool):
    def use(self): return "Hammering"

The Logic of Polymorphism

Polymorphism allows objects of different classes to be treated as objects of a common superclass, with methods behaving differently. Below are 10 samples with step-by-step explanations.

class Shape:
    def draw(self): return "Shape"
class Circle(Shape):
    def draw(self): return "Circle"
class Square(Shape):
    def draw(self): return "Square"
def render(s): print(s.draw())
render(Circle()); render(Square())

class Cat:
    def speak(self): return "Meow"
class Dog:
    def speak(self): return "Woof"
def hear(a): print(a.speak())
hear(Cat()); hear(Dog())

class Car:
    def move(self): return "Driving"
class Bike:
    def move(self): return "Cycling"
def travel(v): print(v.move())
travel(Car()); travel(Bike())

class Sparrow:
    def fly(self): return "Sparrow flying"
class Penguin:
    def fly(self): return "Penguin cannot fly"
def soar(b): print(b.fly())

class BasicCalc:
    def add(self, a, b): return int(a+b)
class ScienceCalc:
    def add(self, a, b): return float(a+b)

class Audio:
    def play(self): return "Playing Audio"
class Video:
    def play(self): return "Playing Video"

class Developer:
    def work(self): return "Coding"
class Designer:
    def work(self): return "Designing"

class Banana:
    def describe(self): return "Yellow"
class Orange:
    def describe(self): return "Round"

class Wrench:
    def use(self): return "Tightening"
class Screwdriver:
    def use(self): return "Screwing"

class PayPal:
    def process(self): return "PayPal Processed"
class Stripe:
    def process(self): return "Stripe Processed"
