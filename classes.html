Module 2: Defining the Structure & Lifecycle

In the realm of Python programming, a class is not merely a block of code—it is the DNA of your software ecosystem.

1. The Object Lifecycle: From Birth to Deletion

Every object in Python undergoes a specific sequence of events managed by the Python Interpreter. Understanding this sequence allows you to manage system resources efficiently.

  • Declaration: The class is defined in the script memory.
  • Instantiation: An object is created using the class blueprint.
  • Initialization: The __init__ method sets the starting state.
  • Operation: The object performs tasks via methods.
  • Destruction: Reclaimed by the Garbage Collector.

2. The Architecture of a Constructor

The __init__ method is technically a "magic method." It is called automatically by Python when you create a new instance of a class.

[Image illustrating a blueprint vs a finished building to explain Class vs Object] 
class Satellite:
    def __init__(self, name, orbit_altitude):
        # Assigning instance-specific data
        self.name = name
        self.altitude = orbit_altitude
        self.status = "Active"

The Importance of self

In Python Architecture, self is a reference to the **current instance** of the class. It is the bridge that connects the data inside an object to the methods that operate on it.

3. Memory Management & Garbage Collection

Python uses Reference Counting. When you assign a variable to an object, the reference count increases. When the count reaches zero, Python reclaims that memory.
