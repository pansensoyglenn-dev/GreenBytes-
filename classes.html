Module 2: Defining the Structure & Lifecycle

In Python, every object has a lifecycle: it is born (instantiated), it performs work (method execution), and it eventually expires (garbage collection). Understanding the syntax of a Class is the first step in managing the lifecycle of your AI agents.

The Anatomy of a Class

A class is defined using the class keyword. Inside, we define two primary things: Attributes (the data the object holds) and Methods (the functions the object can perform).

1. The Constructor: __init__

The __init__ method is a special "dunder" (double underscore) method. It is called automatically when you create a new instance. Its job is to initialize the object's state. In the context of AI, this is where we might load a model configuration or set an API key.

2. The Identity: self

The self parameter represents the specific instance of the class. It is the "internal address" that allows an object to refer to its own attributes. Without self, an object would not have a concept of its own identity or memory.

Practical Implementation

Let's build a basic KnowledgeBase class that acts as an isolated memory unit for an AI agent. Notice how self is used to bind data to a specific instance. 

class KnowledgeBase:
    def __init__(self, topic: str):
        # Initializing the attribute
        self.topic = topic
        self.entries = []

    def add_entry(self, detail: str):
        # Modifying internal state
        self.entries.append(detail)
        print(f"Update: New data added to {self.topic} memory.")

# Instantiation (Creating the Object)
python_kb = KnowledgeBase("Python Programming")
python_kb.add_entry("Classes are blueprints for objects.")

In this example, python_kb is an instance. If we created another instance for "Machine Learning," their entries lists would be entirely separate. This fundamental isolation is what prevents data leakage in production-grade Python applications.
