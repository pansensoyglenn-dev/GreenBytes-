I. Full Source Code

# Vegetable class to encapsulate properties of each vegetable
class Vegetable:
    def __init__(self, name, price_per_kilo, initial_stock):
        self.name = name
        self.price_per_kilo = price_per_kilo
        self.stock = initial_stock  # Initial stock in kilos

    def sell(self, quantity):
        if quantity > self.stock:
            raise ValueError(f"Not enough stock for {self.name}. Available: {self.stock} kg")
        self.stock -= quantity
        return quantity * self.price_per_kilo

    def get_remaining_stock(self):
        return self.stock

# Store class to manage multiple vegetables
class VegetableStore:
    def __init__(self):
        # Dictionary of vegetables with their details (assuming initial stock of 100 kg each)
        self.vegetables = {
            "asparagus": Vegetable("Asparagus", 180, 100),
            "cabbage": Vegetable("Cabbage", 140, 100),
            "onion": Vegetable("Onion", 180, 100),
            "garlic": Vegetable("Garlic", 190, 100),
            "tomatoes": Vegetable("Tomatoes", 160, 100),
            "lettuce": Vegetable("Lettuce", 180, 100),
            "broccoli": Vegetable("Broccoli", 230, 100),
            "cauliflower": Vegetable("Cauliflower", 220, 100),
            "potatoes": Vegetable("Potatoes", 180, 100)
        }

    def list_vegetables(self):
        print("Available Vegetables:")
        for key, veg in self.vegetables.items():
            print(f"{key.capitalize()}: {veg.price_per_kilo} per kg, Stock: {veg.get_remaining_stock()} kg")

    def sell_vegetable(self, choice, quantity):
        choice_lower = choice.lower()
        if choice_lower not in self.vegetables:
            raise ValueError(f"{choice} is not available in the store.")
        total_price = self.vegetables[choice_lower].sell(quantity)
        return total_price, self.vegetables[choice_lower].get_remaining_stock()

# Main function to run the program
def main():
    store = VegetableStore()
    
    while True:
        store.list_vegetables()
        
        choice = input("\nEnter the vegetable you want to buy (or 'exit' to quit): ").strip()
        if choice.lower() == 'exit':
            break
        
        try:
            quantity = float(input(f"Enter the quantity in kg for {choice}: "))
            if quantity <= 0:
                print("Quantity must be positive.")
                continue
            
            total, remaining = store.sell_vegetable(choice, quantity)
            print(f"Total price: {total}")
            print(f"Remaining stock for {choice}: {remaining} kg")
        
        except ValueError as e:
            print(e)
        
        print("\n")

if __name__ == "__main__":
    main()

II. Vegetable Class Breakdown

class Vegetable:
Logical Insight
This line defines a new class named Vegetable. In Python, classes are blueprints for creating objects that bundle data and methods together.
Significance
It sets up the foundation for encapsulation by grouping related attributes (like name, price, and stock) and behaviors (like selling) into a single unit. This promotes modularity and reusability—each vegetable type can be treated as an independent entity without exposing internal details.
How it Works
When executed, Python creates a class object in memory. Objects instantiated from this class will inherit its structure. This is the entry point for object-oriented programming (OOP) principles in the program.
def __init__(self, name, price_per_kilo, initial_stock):
Logical Insight
This is the constructor method, called automatically when a new Vegetable object is created. It initializes the object's state with provided values.
Significance
Encapsulation starts here by binding data (attributes) to the object. It ensures every vegetable instance has a consistent setup, preventing uninitialized states that could lead to errors.
How it Works
When you do Vegetable("Asparagus", 180, 100), Python calls __init__ with self (the new object) and the arguments. It assigns values to instance variables, making them accessible via self.
Attribute Initialization (self.name, self.price_per_kilo, self.stock)
Logical Insight
These lines assign the passed arguments to instance attributes. self.name stores the vegetable's name, self.price_per_kilo its price, and self.stock the available quantity.
Significance
This hides the data within the object (encapsulation's data hiding aspect). External code can't directly tamper with these without going through methods, reducing bugs from invalid modifications.
How it Works
self refers to the current instance, so these become properties of the object. The comment clarifies units, aiding readability. If you try to access veg.stock directly outside the class, it's possible in Python (no strict private access), but the design encourages using methods for interaction.
def sell(self, quantity):
Logical Insight
Defines a method to handle selling a specified quantity of the vegetable.
Significance
This is a key encapsulation feature—it provides controlled access to modify stock. Instead of allowing direct subtraction (which could go negative), it enforces rules, maintaining data integrity.
How it Works
When called like veg.sell(5), it receives quantity and uses self to access the object's data. It performs checks and calculations internally.
if quantity > self.stock:
Logical Insight
Checks if the requested quantity exceeds available stock and raises an error if so.
Significance
Prevents invalid states (e.g., negative stock), embodying encapsulation's role in validation. It makes the program robust and user-friendly by providing informative errors.
How it Works
Compares quantity to self.stock. If true, it creates a ValueError with a formatted string (using f-string for dynamic insertion of values) and throws it, halting execution unless caught.
self.stock -= quantity
Logical Insight
Subtracts the sold quantity from the stock.
Significance
This is the core mutation of encapsulated data—only allowed after passing the validation check, ensuring consistency.
How it Works
Uses the -= operator to decrement self.stock. Since stock is a number (float or int), this arithmetic operation updates the instance's state permanently.
return quantity * self.price_per_kilo
Logical Insight
Calculates and returns the total cost of the sold quantity.
Significance
Separates concerns—method handles both state change and computation, returning only what's needed (total price) without exposing internal stock details unnecessarily.
How it Works
Multiplies quantity by self.price_per_kilo and returns the result. This allows the caller to get the price without direct access to attributes.
def get_remaining_stock(self):
Logical Insight
Defines a getter method to retrieve the current stock level.
Significance
Encapsulates read access to stock. Direct access could be risky if logic changes (e.g., adding units conversion); this method allows future modifications without breaking external code.
How it Works
Simply returns self.stock when called, like veg.get_remaining_stock(). It's a read-only interface, promoting the "tell, don't ask" principle in OOP.
Return Logic Insight
Returns the value of the stock attribute.
Return Significance
Keeps the method simple and focused, aligning with encapsulation by not allowing modification here.
Return How it Works
The return statement sends the value back to the caller, ending the method execution.

V. Overall Program Summary

This program exemplifies encapsulation by:

  • Bundling data and methods in classes (Vegetable for items, VegetableStore for management).
  • Controlling access via methods (e.g., sell() validates before changing state).
  • Hiding internals (e.g., dict structure, direct stock access discouraged). It works as an interactive loop: lists inventory, takes inputs, processes sales with checks, updates states, and repeats. This design ensures data integrity, modularity, and ease of extension (e.g., add discounts via new methods).
