Mastering Attributes

Architecting data persistence in Python Objects.

The Architectural Significance

In modern software engineering, efficiency is driven by state management. Attributes are the Internal State of your system. Mismanaging them leads to "Shadowing" and "Race Conditions" in asynchronous Python applications.

1. Instance Attributes: The Individual State

Instance attributes are owned by a specific object. They are defined inside the __init__ method using the self keyword. This ensures that every object created from a class can hold unique values, isolated from other instances.

class User:
    def __init__(self, username, access_level):
        self.username = username  # Instance Attribute
        self.access_level = access_level

user1 = User("Alice", "Admin")
user2 = User("Bob", "Guest")

2. Class Attributes: The Shared Metadata

Class attributes are defined directly inside the class body. They are shared by all instances of the class. If you change a class attribute, every existing and future instance sees that change simultaneously because they all point to the same memory location for that attribute.

Use Case

Class attributes are ideal for constants (like PI), shared configuration settings, or counters that track how many objects have been instantiated across the entire application lifetime.

3. The Resolution Order (Namespace Lookup)

When you call object.attribute, Python follows a strict lookup hierarchy known as the namespace resolution order:

  1. Search the Instance Namespace (attributes unique to that specific object).
  2. Search the Class Namespace (attributes shared by all instances of that class).
  3. Search the Parent Classes (following the Method Resolution Order or MRO if inheritance is used).

COMMON PITFALL: Mutable Class Attributes

A common mistake is using mutable types (like lists or dictionaries) as class attributes. Because these are shared in memory, an update from one instance leaks into all others, leading to difficult-to-trace bugs.

class Database:
    logs = [] # DANGEROUS: Shared across all instances

db1 = Database()
db1.logs.append("Connection Error")

db2 = Database()
print(db2.logs) # Output: ["Connection Error"] - Unexpected sharing!

Pro Solution: Always initialize mutable attributes inside __init__ to ensure each object gets its own private memory space.
