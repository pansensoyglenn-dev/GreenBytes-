Philosophy of Objects

The Core Idea

Object-Oriented Programming (OOP) allows you to bundle data + behavior that belongs to that data into one logical unit called an object. You then create many similar objects from a reusable blueprint called a class. This approach mirrors real-world logic, where entities aren't just collections of variables, but active participants in a system with their own internal rules and external capabilities.

The Python Mental Model

Experienced Python developers view classes mostly as namespaces that know how to create objects. Objects themselves are bundles of data that pass self as the first argument to their internal functions. Most of the time, the architecture should prioritize behavior and polymorphism over deep, rigid inheritance hierarchies. If inheritance becomes painful, modern practice favors composition and small protocols over massive class trees.

Minimal Implementation

Modern Python OOP utilizes @dataclasses for clean data structure and ABC for formal interfaces. This ensures your code is both readable and strictly defined, which is essential for scaling complex software systems in 2026.

# Data + behavior living together
from dataclasses import dataclass
from abc import ABC, abstractmethod

class Drawable(ABC):
    @abstractmethod
    def draw(self) -> str: pass

@dataclass
class Point:
    x: float
    y: float

class Circle(Drawable):
    def __init__(self, center: Point, radius: float):
        self._center = center # protected by convention
        self._radius = radius

    def draw(self) -> str:
        return f"○ at {self._center}"

shapes = [Circle(Point(0,0), 5)]
for shape in shapes:
    print(shape.draw()) # polymorphic call
